# Proposal: intentional-bug-recommendations

## Why
Durante la auditoría spec vs código del proyecto Food Store se encontraron **12 desviaciones** entre la documentación técnica (`docs/Integrador.txt`) y la implementación real. Estas desviaciones no son bugs críticos — son decisiones de diseño intencionales o simplificaciones que el equipo tomó durante el desarrollo. Este change las documenta como **deuda técnica** con recomendaciones para resolverlas en el futuro.

## What Changes
Ningún cambio de código en este change. Solo documentación y tracking de deuda técnica. Las 12 desviaciones se listan en `tasks.md` con prioridad y change sugerido para cada una.

## Scope
- **Archivos:** 0 (solo artefactos OPSX)
- **Riesgo:** Nulo — solo documentación
- **Dependencias:** Ninguna
